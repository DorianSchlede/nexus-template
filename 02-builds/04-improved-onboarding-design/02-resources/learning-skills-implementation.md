### Part A: Living Knowledge Organism

**Core Concept:**
```
ALL FILES IN THIS SYSTEM ARE EXECUTABLE ‚Äî NOT DOCUMENTATION!

Every .md file, .yaml config, and planning document is designed
to be READ, LOADED, and EXECUTED by AI in conversation.

This is not a static knowledge base. It's a living, breathing
organism that guides you through work.
```

### Part B: The 7 Problems Nexus Solves

1. **AI AMNESIA** - AI forgets everything
   ‚Üí Solution: Memory files auto-load every session

2. **FILE CHAOS** - AI generates files randomly
   ‚Üí Solution: 5-folder system with clear boundaries

3. **INCONSISTENT RESULTS** - AI improvises differently
   ‚Üí Solution: Skills = saved workflows

4. **REPEATED WORK** - Build same workflow multiple times
   ‚Üí Solution: Create skill once, reuse forever

5. **CONTEXT OVERLOAD** - Loading everything slows responses
   ‚Üí Solution: Progressive loading (metadata first)

6. **HARD TO LEARN** - Steep learning curve
   ‚Üí Solution: Optional onboarding, smart defaults

7. **IMMEDIATE EXECUTION** - AI builds before understanding
   ‚Üí Solution: Planning mode ‚Üí separate execution session

### Part C: The 7 Design Principles

1. **INSTRUCTION-DRIVEN** - Python script returns COMPLETE instructions, AI follows exactly
2. **YAML-DRIVEN AUTO-DETECTION** - Everything has metadata describing when to load
3. **SKILL-FIRST EXECUTION** - Skills have priority over builds
4. **PROGRESSIVE DISCLOSURE** - Load minimum at start, more just-in-time
5. **STATE IN DATA FILES** - System state tracked in YAML and checkboxes
6. **CONTEXT PRESERVATION** - Nothing lost between sessions
7. **CONCRETE BEFORE ABSTRACT** - Experience first, explanation after

### Part D: System Pitfalls

**Pitfall #1: Builds Instead of Skills**
```
WRONG: weekly-report-week-1/, weekly-report-week-2/
RIGHT: weekly-report/ # ONE skill
```

**Pitfall #2: Skipping Close-Session**
Without "done": Progress not saved, learnings not captured

**Pitfall #3: Over-Organizing**
WRONG (day 1): 20 nested empty folders
RIGHT (day 1): 3-5 essential folders

### Part E: AI Collaboration Patterns

**Pattern #1: False Progress**
- AI: "Done! Files created." (But they don't exist)
- Detection: "Show me the files"

**Pattern #2: Incomplete Reads**
- AI reads 100 of 500 lines, misses critical info
- Detection: "Did you read ENTIRE file?"

**Pattern #3: Over-Engineering**
- You want simple, AI builds framework
- Detection: "Can this be simpler?"

### Part F: Expert Techniques

**Technique #1: "Yes, And..." Collaboration**
Evaluate AI additions, don't auto-accept/reject

**Technique #2: Intelligent Friction**
AI asks clarifying questions to prevent wasted effort

**Technique #3: The Planning Pattern**
```
Session 1 (Planning): Create structure, fill docs, NO IMPLEMENTATION
Session 2 (Execution): Load plans, execute with minimal intervention
```

### Part G: Mastery Check Quiz

**Q1:** Creating "expense-report-jan", "expense-report-feb"... Fix?
- Answer: ONE "expense-report" skill

**Q2:** AI says "Done! All files created." Response?

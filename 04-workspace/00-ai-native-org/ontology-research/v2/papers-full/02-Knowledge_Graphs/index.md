  items:
    - "Node: Represents entities of interest in a knowledge graph (e.g., cities, events, people)"
    - "Edge: Represents relations between entities in a knowledge graph"
    - "Entity: Real-world object or concept represented by nodes in the domain graph"
    - "Relation: Binary relationship between entities represented by edges"
    - "Class: Grouping of similar entities (e.g., Event, City, Place, Venue)"
    - "Property: Edge label denoting a type of relation (e.g., venue, city, flight)"
    - "Individual: Specific entity instance in an ontology (e.g., Santiago, EID16)"
    - "Literal: Datatype value node (strings, integers, dates)"
    - "Blank Node: Anonymous/existential node without assigned identifier"
    - "Named Graph: Graph with an associated identifier for provenance/context"
    - "Directed Edge-Labelled Graph: Tuple G = (V, E, L) where V is nodes, L is edge labels, E is edges"
    - "Property Graph: Tuple G = (V, E, L, P, U, e, l, p) with node/edge properties"
    - "Heterogeneous Graph: Tuple G = (V, E, L, l) with node type labeling function"

entity_definitions:
  items:
    - "Knowledge Graph: A graph of data intended to accumulate and convey knowledge of the real world, whose nodes represent entities of interest and whose edges represent relations between these entities"
    - "Data Graph: A collection of data represented as nodes and edges using one of the graph-based data models"
    - "Ontology: A concrete, formal representation of what terms mean within the scope in which they are used"
    - "Interpretation: Composed of a domain graph and a mapping from data graph terms to domain graph terms"
    - "Model: An interpretation that satisfies a graph (satisfies all semantic conditions)"
    - "Entailment: One graph entails another if all models of the first are also models of the second"
    - "Schema: High-level structure and/or semantics that a graph follows (semantic, validating, or emergent)"
    - "Semantic Schema: Defines meaning of high-level terms, enabling inference (RDFS, OWL)"
    - "Validating Schema: Defines constraints for validation (SHACL, ShEx shapes)"
    - "Emergent Schema: Latent structures automatically extracted as quotient graphs"

entity_relationships:
  items:
    - "subclass-of: Hierarchical relationship between classes (City subclass-of Place)"
    - "subproperty-of: Hierarchical relationship between properties (venue subproperty-of location)"
    - "domain: Associates a property with the class of its subject nodes"
    - "range: Associates a property with the class of its object nodes"
    - "type: Asserts class membership of an individual"
    - "same-as: Identity link stating two nodes refer to the same entity"
    - "different-from: States two nodes refer to different entities"
    - "inverse-of: Relates properties that are inverses (venue inverse-of hosts)"
    - "equivalent-class: States two classes have the same instances"
    - "disjoint-with: States two classes have no common instances"
    - "transitive: Property where x-p-y and y-p-z implies x-p-z"
    - "symmetric: Property where x-p-y implies y-p-x"
    - "functional: Property with at most one value per subject (many-to-one)"
    - "inverse-functional: Property with at most one subject per value (one-to-many)"
    - "property-chain: Relates a property to a sequence of properties"

# Knowledge Graphs - Comprehensive Survey

This paper provides a comprehensive tutorial-style introduction to knowledge graphs, covering foundational concepts through advanced techniques for representing, querying, reasoning, and learning over graph-structured knowledge.

## Paper Overview

- **Source**: 02-Knowledge_Graphs.pdf
- **Total Chunks**: 15 chunks analyzed
- **Analyzed**: 2025-12-31
- **Authors**: Aidan Hogan et al. (18 authors from multiple institutions)
- **Type**: Comprehensive tutorial/survey paper on knowledge graphs

## Key Topics Covered

1. **Data Graphs**: Graph data models (directed edge-labelled, property graphs, heterogeneous graphs) and query languages (SPARQL, Cypher, graph patterns)

2. **Schema, Identity, Context**: Semantic schema (RDFS), validating schema (SHACL, ShEx), emergent schema, persistent identifiers, context representation

3. **Deductive Knowledge**: Ontologies (OWL), Description Logics, inference rules, materialisation, query rewriting

4. **Inductive Knowledge**: Graph analytics, knowledge graph embeddings, graph neural networks, symbolic learning (rule mining, axiom mining)

5. **Creation and Enrichment**: Text extraction, entity linking, relation extraction, knowledge acquisition

6. **Quality and Refinement**: Quality dimensions, error detection, link prediction, entity resolution

7. **Publishing**: Linked Data principles, knowledge graph publication standards

8. **Applications**: Open knowledge graphs (DBpedia, Wikidata, YAGO), enterprise knowledge graphs (Google, Bing, Facebook)

## Paper Type

Tutorial/Survey paper providing comprehensive coverage of knowledge graph concepts, techniques, and applications.

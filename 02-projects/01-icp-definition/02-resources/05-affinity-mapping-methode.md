# Affinity Mapping Methode

> **Source**: UX Research Best Practices (December 2024)
> **Usage**: Lade diese Ressource für Interview-Analyse

---

## Was ist Affinity Mapping?

A collaborative method to organize and make sense of large amounts of information by thematically analyzing qualitative research data.

**Purpose**: Identifies patterns that might not be apparent when looking at individual data points.

---

## Prozess (5 Steps)

### 1. SAMMELN
```
- Alle Interview-Notes auf einzelne Karten/Post-its
- Eine Beobachtung pro Karte
- Direkte Zitate verwenden
- Jede Karte = eine Idee, Beobachtung, oder Quote
```

**Tools**: Miro, FigJam, physical Post-its, Notion

### 2. CLUSTERN
```
- Ähnliche Beobachtungen gruppieren
- Keine vordefinierten Kategorien!
- Natürliche Cluster entstehen lassen
- Move cards around until patterns emerge
```

**Tip**: Do this silently first, then discuss as a team.

### 3. BENENNEN
```
- Jedes Cluster mit Theme benennen
- Themes sollten Insights sein, nicht Kategorien
- z.B. "Frustration mit Agent-Qualität" statt "Feedback"
- Each label should tell a story
```

**Good Label**: "Users spend 2+ hours daily fixing AI errors"
**Bad Label**: "AI Problems"

### 4. PRIORISIEREN
```
- Welche Cluster sind größer? (mehr Mentions)
- Welche haben höhere Intensität?
- Welche korrelieren mit ICP-Fit?
- Which patterns appear across multiple interviews?
```

### 5. VALIDIEREN
```
- Patterns gegen quantitative Daten prüfen
- Ausreißer identifizieren
- Hypothesen formulieren
- Test patterns in subsequent interviews
```

## Saturation Point

| Interview Count | What to Expect |
|-----------------|----------------|
| 5 Interviews | First hypotheses, early patterns |
| 10-15 Interviews | Clear patterns emerge |
| 20-25 Interviews | Saturation (wenig neue Insights) |
| 30 Interviews | Statistical significance |

**Saturation Indicators**:
- Same patterns appear repeatedly
- Few surprising new insights
- Interviewees confirm what you already know

---

## What to Look For

| Pattern-Typ | Fragen |
|-------------|--------|
| **Common Pains** | Welche Probleme tauchen immer wieder auf? |
| **Trigger Events** | Was führt zur Suche nach Lösungen? |
| **Success Metrics** | Wie definieren sie Erfolg? |
| **Decision Criteria** | Was ist ihnen bei der Auswahl wichtig? |
| **Blockers** | Was hält sie vom Kauf ab? |
| **Workarounds** | Wie lösen sie das Problem heute? |
| **Emotional Drivers** | Was frustriert sie? Was begeistert sie? |

## Affinity Mapping vs Thematic Analysis

| Aspect | Affinity Mapping | Thematic Analysis |
|--------|------------------|-------------------|
| **Focus** | Grouping ideas based on relationships | Identifying and interpreting patterns |
| **Timing** | Early stages of research | After data collection |
| **Output** | Visual map showing clusters | Set of defined themes |
| **Collaboration** | Highly collaborative, group work | Often individual |

**Recommendation**: Use affinity mapping first (visual, collaborative), then thematic analysis for deeper interpretation.

---

## Avoiding Bias

### Confirmation Bias
- Don't prioritize data that confirms your existing beliefs
- Actively look for contradicting evidence

### Recency Bias
- Don't overweight recent interviews
- Review all data equally

### Groupthink
- Do silent clustering before group discussion
- Encourage diverse perspectives
